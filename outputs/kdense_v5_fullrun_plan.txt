A) Must-fix decisions (primary vs sensitivity analyses)

Primary analysis must address:
1.  **Null Contradiction:** Re-run null model permutations. The null AUROC must consistently center around 0.5. The existing reported values (0.38-0.41 vs 0.49) are inconsistent and indicate a fundamental error in null definition or calculation. A correctly implemented null should show AUROC ~0.5. The lower reported values are likely artifacts of miscalculation.
2.  **Leakage Concern (ComBat):** Re-implement ComBat application. It must be fit *only* on the training dataset within each cross-validation fold. This is non-negotiable for publishable work.
3.  **AMP-AD Integration:** Re-evaluate and implement direct model integration of AMP-AD data where scientifically justified (e.g., as a feature set). If purely contextual, this must be explicitly stated and justified in the methods.
4.  **Novelty Framing:** Revise the core claim of novelty to be more conservative and precise, focusing on specific methodological contributions or unique findings rather than broad overstatements.

Sensitivity analyses will include:
1.  Comparison of `source_plus_target_raw` vs. the new, correctly implemented `source_plus_target_combat` to assess the isolated impact of ComBat correction without leakage.
2.  Performance across the new feature sets (`DE-only`, `Agora-only`, `DE∩Agora`) to understand the contribution of different information sources.

**Null Definition:**
*   For `null_label_permutation_avg100_redefined`: Labels are permuted *within* the training set for each cross-validation fold, ensuring that the feature-label associations are completely randomized while preserving sample sizes and class distributions within folds. Each permutation involves randomly shuffling the disease/control labels for all samples within a given training split, followed by model training and prediction on the corresponding test set. This process is repeated 100 times to generate an average null AUROC distribution.
